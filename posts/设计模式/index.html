单例模式#

单例模式的作用：无论对象被实例化多少次，确保全局只有一个实例存在。

应用：全局唯一性配置、数据库连接对象、文件访问对象

1. 饿汉式单例#

这种单例对象在包加载时即被创建，因此叫饿汉式。当单例实例化时要初始化的内容太多，会造成加载时间过长，不推荐使用饿汉式。

var instance = &singleton{}

func GetSingleton() *singleton {
    return instance

2. 懒汉式单例#

与上面饥渴的饿汉式相比，懒汉式是在第一次使用实例时被创建。

package singleton

type singleton struct {}

var instance *singleton

func GetSingleton() *singleton {
    if instance == nil {
        instance = &singleton{}
    }
    return instance
}

将对象实例化延迟到第一次调用GetSingleton()方法时。此外，instance == nil在多个goroutine调用时无法确保并发安全。

3. 支持并发的懒汉单例#

package singleton
import "sync"
var (
    instance *singleton
    mu sync.Mutex
)
func GetSingleton() *singleton {
    mu.Lock()
    defer mu.Unlock()

    if instance == nil {
        instance = &singleton{}
    }
    return instance
}

加锁支持了并发安全，但频繁地加锁解锁又带来了性能问题。

4. 双重锁定（Double-Check Locking）#

package singleton
import "sync"
var (
    instance *singleton
    mu sync.Mutex
)
func GetSingleton() *singleton {
    if instance == nil {  // 外层的 instance == nil 判断是为了提高程序的执行效率，免去原来每次调用 GetSingleton () 都上锁的操作，将加锁的粒度更加精细化。
        mu.Lock()
        defer mu.Unlock()
        instance = &singleton{}
    }
    return instance
}

5. 最终的Gopher样式单例#

go将上面的实现封装到了sync/once中

 1
 2
 3
 4
 5
 6
 7
 8

10
11

package singleton
import "sync"
var (
    instance *singleton
    once sync.Once
)
func GetSingleton() *singleton {
    once.Do(func() {
        instance = &singleton{}
    })
    return instance
}

Once 是一个结构体，在执行 Do() 方法的内部通过 atomic 操作和加锁机制来保证并发安全，且 once.Do 能够保证多个 goroutine 同时执行时 &singleton {} 只被创建一次。 Once 内部实现跟上面使用的双重锁定机制非常类似，只不过把 instance == nil 换成了 atomic 操作。

type Once struct {
    done atomic.Uint32
    mu Lock
}

选项模式#

Go 语言没有构造函数，一般通过定义 New 函数来充当构造函数。然而，如果结构有较多字段，要初始化这些字段，有很多种方式，但有一种方式认为是最好的，这就是函数式选项模式（Functional Options Pattern）。

package main

import (
    "log"
    "server"
)

type Server struct {
    host string
    port int
}

func New(host string, port int) *Server {
    return &Server{host, port}
}

func (s *Server) Start() error {}

func main() {
    srv := New("localhost", 1234)
    if err := srv.Start(); err != nil {
        log.Fatal(err)
    }
}

但如果要扩展 Server 的配置选项（比如我想加一个timeout和maxconn），如何做？ 通常有三种做法：

  • 为每个不同的配置选项声明一个新的构造函数
  • 定义一个新的 Config 结构体来保存配置信息
  • 使用 Functional Option Pattern

func New(host string, port int) *Server {
  return &Server{host, port, time.Minute, 100}

}

func NewWithTimeoutAndMaxConn(host string, port int, timeout time.Duration, maxConn int) *Server {

type Option func(*Server)

func New(options ...Option) *Server {
    svr := &Server{}

    for _, f := range options {
        f(svr)
    }
    return svr
}

func WithHost(host string) Option {
    return func(s *Server) {
        s.host = host
    }
}

func WithPort(port int) Option {
    return func(s *Server) {
        s.port = port
    }
}

func WithTimeout(timeout time.Duration) Option {
  return func(s *Server) {
    s.timeout = timeout
  }
}

func WithMaxConn(maxConn int) Option {
  return func(s *Server) {
    s.maxConn = maxConn
  }
}

)

func main() {
    svr := New(
        WithHost("localhost"),
        WithPort(8080),
        WithTimeout(time.Minute),
        WithMaxConn(120),
    )
    if err := svr.Start(); err != nil {
        log.Fatal(err)
    }
}
